# Pinky Market

Next.js фронт (`front/`) + Go бэкенд (`backend/`) + Postgres. Деплой — Dokploy, фронт и бэкенд как отдельные сервисы.

## ВАЖНО: не запускать билды локально

**НЕ запускай `npm run build`, `next build`, `go build` и т.п. локально.** Владелец выгружает прод-версию напрямую в Dokploy — сборка происходит там. Локальные билды не нужны, тратят время и засоряют рабочую папку (`.next/`, `node_modules/`). На машине нет Go и Docker — это нормально, ставить их не надо.

Проверка корректности кода — только чтением/ревью, без сборки.

## Прочее

- Миграции БД применяются автоматически при старте бэкенда (`CREATE TABLE IF NOT EXISTS` в `backend/db.go`)
- Фронт ходит в API через прокси `/api/*` (`front/app/api/[...path]/route.ts`), бэкенд-адрес берётся из runtime `API_URL`
- Аккаунты анонимные (ник + пароль, без email), токен в localStorage
